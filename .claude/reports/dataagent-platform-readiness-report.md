# DataAgent 前端、测试与平台完备度调研报告

生成时间：2026-05-10  
范围：前端接入、测试体系、CI/CD、配置治理、可观测性

---

# 一、当前已具备的外围能力

## 1. 基础 Maven 工程能力
已具备：
- Spring Boot 构建
- Spotless 校验
- `mvn verify`
- GitHub Actions CI 基线

## 2. 后端接口形态已成型
当前已具备：
- Agent 接口
- 数据源管理接口
- 表语义接口
- 列语义接口
- 关系语义接口

这说明前端如果接入，并不需要等后端 API 再从零设计。

---

# 二、当前明显缺失

## 1. 前端工程几乎未落地
当前仓库中：
- `data-agent-frontend` 仅见 README
- 未见页面源码
- 未见接口调用代码
- 未见构建脚本

这意味着：
- 当前更像纯后端工程 + 预留前端目录
- 产品体验层仍未形成

## 2. 测试体系缺失
当前未见：
- `src/test`
- controller 单测
- service 单测
- 数据库相关测试
- Agent 主链路测试

## 3. CI/CD 很薄
当前只有：
- `mvn verify`

没有：
- 制品发布
- 镜像构建
- 环境部署
- 回滚
- 质量门禁

## 4. 配置治理薄弱
当前：
- 无 profile 分层
- 无 secrets 管理
- 无配置校验聚合

## 5. 可观测性几乎为空
当前未见：
- actuator
- micrometer
- tracing
- metrics
- error tracking
- dashboard

---

# 三、要成为完善 DataAgent 必须补什么

## P0
1. 前端真实工程
2. 测试目录与测试基线
3. 配置安全治理

## P1
4. 可观测性（health / metrics / 日志）
5. 前后端联调闭环
6. 基本部署链路

## P2
7. 生产级 CI/CD
8. 监控告警
9. 错误追踪

---

# 四、建议的最小闭环

如果目标是先形成“一个可演示可联调的完整 DataAgent”，建议最小闭环是：

1. 补前端工程骨架
2. 接上 Agent 聊天页
3. 接上数据源管理页
4. 接上语义管理页
5. 后端补最基本测试
6. CI 保证能构建、校验、跑测试

---

# 五、结论

当前项目离“完整平台产品”的差距，不在后端接口数量，而在：
- 前端不存在
- 测试不存在
- 平台能力薄
- 配置治理弱
- 可观测性为空

所以从产品完备度看，前端、测试和运维能力是下一层必须补的能力。
