# DataAgent 部署与生产就绪度调研报告

生成时间：2026-05-10  
范围：部署、配置、环境、最小生产可用能力

---

# 一、当前现状

当前已具备：
- Spring Boot 可运行后端
- Maven 构建
- GitHub Actions `mvn verify`
- 动态数据源能力
- Agent 会话 MySQL 持久化

当前仍缺：
- 配置分层
- secrets 管理
- 健康检查
- 鉴权与最小权限控制
- 制品 / 镜像发布链路
- 回滚能力

---

# 二、从当前到最小生产可用还缺什么

## 1. 配置分层
当前 `application.properties` 仍是单环境配置，且包含明文 DB 凭据。

必须补：
- `dev/test/prod` profile
- 环境变量注入
- 敏感配置脱敏

## 2. 健康检查
必须补：
- 应用健康检查
- 数据源健康检查
- Agent 依赖模型可用性检查（至少基础可观测）

## 3. 鉴权与最小权限
必须补：
- 管理接口鉴权
- 最小权限控制
- session / datasource / query 审计

## 4. 发布链路
当前只有 verify，没有：
- jar/release 制品发布
- 镜像构建
- 环境部署
- 回滚流程

## 5. 会话与管理库初始化
必须明确：
- `agentscope_sessions` 表初始化策略
- 管理库 schema 初始化策略

---

# 三、最小生产环境清单

建议最小生产环境至少包含：

1. JDK 17 运行环境
2. 1 个 Spring Boot 服务实例
3. 1 个 MySQL 管理库
4. 外部 LLM 配置与 API Key 注入
5. 独立 prod 配置
6. HTTPS / 反向代理
7. 基础认证鉴权
8. 基础日志与健康检查
9. 制品或容器发布流程

---

# 四、优先级建议

## P0
1. 配置治理
2. 健康检查
3. 认证鉴权
4. 管理库初始化与 migration
5. 发布链路雏形

## P1
6. 镜像化
7. 多环境发布
8. 回滚流程

---

# 五、结论

当前项目可以跑在开发环境，但离“最小生产可用”还差一层明显的部署与配置治理能力。

> 如果要继续往产品和平台推进，部署与配置治理必须尽早纳入主路线，而不是等前端做完再补。
