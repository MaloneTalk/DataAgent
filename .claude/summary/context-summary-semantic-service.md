## 项目上下文摘要（DataAgent 当前现状）
生成时间：2026-05-10  
**最后更新：2026-05-17**

### 1. 当前已落地的核心能力

- **语义层**：表语义、列语义、关系语义三层，分页/搜索/排序/批量重置均已落地，已接入 Agent 主链路。
- **Agent 主链路**：同步聊天、SSE 流式聊天、Session 持久化、`get_tables → get_table_schema → execute_sql` 工具编排闭环，`maxIters=16`。
- **语义合并引擎**：`SemanticCatalog` 提供请求级缓存的物理+语义合并上下文（`CatalogContext`），`SchemaReader` 提供 TTL 60s 本地缓存。
- **数据源与执行链**：数据源 CRUD、动态连接池、物理 schema 读取（含复合外键聚合）、只读 SQL 执行。
- **工程基线**：Maven 构建、Spotless 校验、最小 GitHub Actions CI。

### 2. 当前系统定位

> **一个已经具备完整语义层管理能力的查数助手后端骨架**

而不是：
- 纯语义原型
- 完整的产品化 DataAgent
- 成熟的生产级数据代理平台

### 3. 与上次评估相比，已完成的改进

| 项目 | 旧状态 | 当前状态 |
|---|---|---|
| 表/列分页+搜索+排序（C1/C2） | 进行中 | **已完成** |
| 关系分页+搜索+排序（C3） | 进行中 | **已完成** |
| 批量重置（C4） | 部分完成 | **已完成**（表/列批量重置 + 关系批量删除） |
| 包结构收口（D1） | 未开始 | **已完成**（全部迁入 `service.semantic`） |
| SemanticCatalog | 不存在 | **已引入**（请求级合并缓存） |
| 复合外键聚合 | 不支持 | **已实现** |

### 4. 当前最主要的整体缺口

除语义层自身的缺口外，DataAgent 仍明显缺：
- **Agent 产品能力**：显式 datasource 路由、结构化事件流、异步长查询、查询历史
- **数据源与执行安全**：健康检查、密码治理、更强 SQL 校验、查询审计
- **前端与交互层**：真实前端工程、聊天页、语义管理页、数据源管理页
- **测试与平台能力**：`src/test`、可观测性、配置治理

### 5. 当前最值得优先推进的方向

#### P0
1. **测试基线**（`src/test` 目录不存在，核心链路无自动化回归）
2. **migration / 升级方案**（缺已有环境升级路径）
3. **显式 datasource 路由**（多 active 取第一个，存在跨库误查风险）

#### P1
4. 前端真实工程
5. 结构化 SSE 事件流
6. 数据源安全治理与查询审计

#### P2
7. 高级字段语义（display name, semantic type, time role, enum schema）
8. 指标语义层
9. 治理层（审计 / 版本 / draft / published）

### 6. 当前关键风险点

- **多数据源路由不明确**：active datasource 取第一个，存在跨库误查风险
- **SQL 执行安全边界仍偏基础**
- **前端未落地，产品闭环不成立**
- **测试目录不存在，核心链路缺自动化回归保护**
- **流式输出仍是文本流，产品化表达能力弱**
